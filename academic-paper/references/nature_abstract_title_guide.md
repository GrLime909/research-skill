# Nature-Style Abstract & Title Guide

> **Purpose**: Supplementary abstract and title writing patterns adapted from nature-writing. The existing `abstract_writing_guide.md` provides a structured 5-component model (Background → Purpose → Method → Findings → Implications) suited to social-science journals. This file adds an unstructured, narrative-flow alternative common in Nature-family journals.

## Abstract — Nature-Style Unstructured Paragraph

### Default Flow

`context / problem -> gap -> approach -> key result -> implication -> boundary`

### Paragraph Movement

1. **Field-scale context or problem** — 1 sentence
2. **Why current routes do not fully solve it** — 1 sentence
3. **What this paper introduces or demonstrates** — 1 sentence
4. **The strongest result, with quantitative or comparative support** — 1-2 sentences
5. **The mechanism, workflow, or practical consequence** — 1 sentence
6. **Bounded implication** — 1 sentence

### Variants for Technical / Method-Heavy Papers

- **Challenge → contribution** — name the bottleneck, state the move
- **Challenge → insight → contribution** — add the conceptual insight between bottleneck and method
- **Multiple contributions** — list contributions after the key result

### Diagnostics

| Symptom | Likely issue |
|---------|-------------|
| Begins with "Here, we" | Missing context — need a background sentence first |
| Ends with a broad promise | Needs scope control — end with a bounded implication |
| No number, comparison, or concrete test | Feels ungrounded — add quantitative support if available |

### Drafting Discipline

- Keep it compact. Cut sentences that re-summarise background the title already implies
- Include quantitative or comparative detail when the user provided it. Do not invent numbers
- End with what the work **enables**, not generic importance
- Write the abstract **last**, when Results and Discussion are stable

### Comparison with the Structured IMRaD Abstract

The existing `abstract_writing_guide.md` gives a **structured 5-component model** (Background → Purpose → Method → Findings → Implications with explicit labels). The Nature-style pattern is **unstructured prose** — same content, no section headings, tighter narrative arc. Pick based on the target journal's requirements.

## Title

### Default Pattern

`system / object + action / capability + application or consequence`

A concrete combination of three elements.

### Drafting Rules

- **Tell the reader what to expect** — not a slogan
- **Be searchable** — include terms a researcher would search for
- **Be substantiated by data** — if a number or quantitative word appears, verify it against the manuscript
- **Create curiosity without sacrificing credibility** — a hook is only acceptable if the claim remains fully defensible

### Things to Avoid

| Pattern | Problem |
|---------|---------|
| Grant-style aims | "Toward …", "A study of …" — vague, says what you intend not what you found |
| Overbroad field claims | "A new era of …", "Revolutionizing …" — unsupported |
| Question titles | Only if the paper genuinely answers the question |
| Unfamiliar jargon | A non-specialist in the target audience should recognise the terms |

### When Asked for Alternatives

Generate 3-5 candidates spanning:
- **Declarative** (the finding as a statement)
- **Finding-led** (the result first)
- **Question** (rarely, and only if the paper answers it)

Mark the most defensible one. Verify any quantitative claim in each candidate.
